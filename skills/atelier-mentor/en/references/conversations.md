# When to start a new conversation, and how to continue

When the question sounds like: "should I open a new conversation or keep
going here?", "how do I pick this back up without re-explaining everything?"

**Long conversations degrade; fresh ones focused on one job perform.** Split
a project's phases across conversations on purpose, rather than stacking
everything into one that keeps growing.

**Hand off with a document, not memory.** A good handoff opens with what the
reader needs to know first, then: what's done, what's next, constraints to
remember, and which skill to use next. That's exactly what `atelier`'s relay
does.

**Give every recurring workspace a reading order.** State which documents to
load first so a fresh session starts oriented, instead of rediscovering
context every time.

**Typical next practice:** if the current conversation has already run long
or shifted topic, offer the relay now rather than waiting to be asked.
