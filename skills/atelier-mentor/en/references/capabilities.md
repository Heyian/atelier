# "Can Claude do X on my setup?"

Never answer this kind of question from memory — see the "Capability
questions" workflow in `SKILL.md` first: verify against `sources.md` and cite
what you find.

General landmarks that tend to stay true over time (not a fixed feature
list):

- Connector-based work (SharePoint, Canva, CRMs) runs in Cowork and on
  claude.ai — the same surfaces the executive already uses.
- Cowork desktop exists for macOS and Windows; capability details shift
  quickly — verify against current sources before promising a workflow.
- A skill has to be enabled on the claude.ai account to be present in Cowork
  sessions.

These three landmarks guide the check, they don't replace it: the exact
current list of supported connectors, which plans include them, and which
file types they handle must always come from a source in `sources.md`, never
from this file.
